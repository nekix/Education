Амортизационная оценка сложности метода Push складывается из операций добавление элемента и расширения структуры List<T>, лежащей в основе данной реализации стека. List реализован как динамический массив, который при переполнении расширяется в два раза. Стоимость расширения массива равна размеру масссива. Стоимость вставки одного элемента равна 1. При вызове push без расширения массива стоимость будет равняться стоимости вставки одного элемента = 1. При вызове push с расширением массива стоимость будет равняться стоимости расширения массива + стоимость вставки одного элемента. При вставке N элементов в пустой стек получаем сложность из сложностей добавления N - 1 элементов без расширения и одного элемента с расширением при размере массива N - 1. Получем общую сложность N - 1 + (N - 1 + 1) = 2N + 1 = 2N. При делении на N полученаем итоговую амортизированную стоимость O(2) = O(1).

В данной реализации сложность метода Peek складывается из операции удаления элемента из списка, а фактически из массива. В таком случае сложность составляет O(1).
