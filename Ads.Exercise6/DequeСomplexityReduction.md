Для понижения (выравнивания) сложности AddHead/RemoveHead и AddTail/RemoveTail методов деки можно использовать в качестве внутренней структуры двунаправленный связный список, позволяющий нам добавлять и удалять элементы за O(1).

Для этого необходимо наличие в связном списке методов удаления/добавления из головы/хвоста, которые сразу бы обрабатывали соответствующие ноды, без необходимости “пробегаться” по всем нодам с целью их поиска. Этим критериям удовлетворяет, например встроенный в стандартную библиотеку C# LinkedList<T>.

В таком случае методы деки будут оберткой над связным списком.
