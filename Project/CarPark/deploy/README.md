# Скрипты развертывания CarPark

## deploy.sh

**Описание:** Скрипт развертывания проекта CarPark на VPS. Устанавливает зависимости, клонирует репозиторий (только поддиректорию проекта), проверяет .env, запускает Docker Compose.

**Использование:** `./deploy.sh [опции]`

**Опции:**
- `-p, --path PATH`: Путь к проекту (по умолчанию /opt/carpark)
- `-m, --mode MODE`: Режим развертывания (minimal или demo, по умолчанию minimal)
- `-g, --git-repo URL`: URL Git-репозитория для клонирования
- `-e, --env-file FILE`: Путь к файлу .env (по умолчанию .env)
- `--help`: Показать справку

**Предварительные требования:**
- Сервер Ubuntu/Debian
- Доступ sudo
- Файл .env с переменными окружения

**Пример:** `./deploy.sh -g https://github.com/user/repo.git -m demo`

## local-deploy.sh

**Описание:** Скрипт локального развертывания. Копирует .env на сервер, скачивает и запускает deploy.sh удаленно.

**Использование:** `./local-deploy.sh -h HOST [опции] [-- [аргументы для deploy.sh]]`

**Опции:**
- `-h, --host HOST`: IP или hostname сервера (обязательно)
- `-u, --user USER`: SSH пользователь (по умолчанию root)
- `-p, --port PORT`: SSH порт (по умолчанию 22)
- `--help`: Показать справку

**Предварительные требования:**
- SSH-доступ к серверу с паролем
- Файл .env в текущей директории
- Аргументы для deploy.sh передаются после --

**Примечание:** Скрипт запросит пароль при подключении к серверу.

**Пример:** `./local-deploy.sh -h 192.168.1.100 -- -m demo -g https://github.com/user/repo.git`
